Ice Cubes gives you full control over the app's appearance through a flexible theme system. You can choose from 14 curated built-in color sets spanning dark and light variants, or override individual colors to create a completely custom look. All theme choices persist across launches via `@AppStorage` and can optionally follow the system's light/dark preference.

## Built-in color sets

## Applying a color set

Call `Theme.applySet(set:)` to switch to any built-in color set at runtime. This updates `selectedSet`, then calls `setColor(withName:)` to write the individual colors into `@AppStorage`.

```swift
// Switch to the Nemesis dark theme
Theme.shared.applySet(set: .nemesisDark)

// Switch to Desert light
Theme.shared.applySet(set: .desertLight)
```

`setColor(withName:)` resolves the matching `ColorSet` from `Theme.allColorSet`, then assigns `tintColor`, `primaryBackgroundColor`, `secondaryBackgroundColor`, and `labelColor` on the shared `Theme` instance.

## Creating a custom theme

When `followSystemColorScheme` is disabled, you can override each color independently using the color pickers in **Settings › Display**. The four editable properties map directly to `Theme` stored properties:

| Property | `@AppStorage` key | Description |
|---|---|---|
| `tintColor` | `tint` | Accent color for interactive elements |
| `primaryBackgroundColor` | `primaryBackground` | Main timeline background |
| `secondaryBackgroundColor` | `secondaryBackground` | Cards, sidebars, form rows |
| `labelColor` | `label` | Primary text color |

Any change to `tintColor`, `primaryBackgroundColor`, or `labelColor` automatically recalculates `contrastingTintColor` so that text on tinted buttons remains legible.

<Note>
Custom color overrides are disabled while `followSystemColorScheme` is `true`. Toggle it off in **Settings › Display › Theme** to unlock the color pickers.
</Note>

## Following the system color scheme

The `followSystemColorScheme` property (stored under the `followSystemColorSchme` key for legacy reasons) instructs Ice Cubes to automatically switch between the dark and light variants of the currently selected color set pair whenever iOS dark mode changes.

```swift
// Enable automatic light/dark switching
Theme.shared.followSystemColorScheme = true

// Lock to a specific scheme regardless of system setting
Theme.shared.followSystemColorScheme = false
Theme.shared.applySet(set: .nemesisDark)
```

When this is enabled, the color pickers in Display Settings are disabled and shown at 50 % opacity.

## Alternate app icons

Ice Cubes includes more than 40 alternate app icons, selectable from **Settings › App Icon**. Icons are contributed by the community and grouped by designer in `IconSelectorView`. The icon assets live in `AlternateIcons/` at the repository root and are registered as `AppIconAlternate1` through `AppIconAlternate49`.

Icons are applied at runtime using `UIApplication.shared.setAlternateIconName(_:)`. The primary icon is restored by passing `nil`.
